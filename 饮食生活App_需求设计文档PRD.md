# 饮食生活 App · 需求设计文档（PRD）

| 项目 | 内容 |
|---|---|
| 产品名称 | 待定（暂称"饮食生活助手"） |
| 文档版本 | v1.0 |
| 日期 | 2026-07-20 |
| 平台 | iOS / Android 移动端 |

---

## 1. 产品概述

### 1.1 定位
一个帮用户**解决"今天吃什么"、管理热量、坚持轻断食**的饮食生活助手，用好友互勉机制提升坚持率。

### 1.2 目标用户
- 有轻度减脂/健康饮食需求的年轻上班族、学生
- 每天纠结"吃什么"的选择困难人群
- 在尝试 16+8 等轻断食、需要监督坚持的人

### 1.3 核心价值
- **决策快**：外卖一键随机，秒解决吃什么
- **好坚持**：断食计时降低放弃率；V1.5 再加入好友互勉
- **省心记**：拍照识别热量，记录零负担

---

## 2. 功能架构总览

完整产品路线包含 5 个业务模块，但 **V1.0 首发底部导航只提供 4 个一级入口**：今天吃什么、拍照识别、轻断食、我的。好友模块仅属于 V1.5，V1.0 的导航、商店截图和功能说明不得出现不可用的好友入口。

| 版本 | 导航 | 模块 | 核心功能 |
|---|---|---|---|
| V1.0 | 🍽 今天吃什么 | 吃什么 | 外卖模式(纯随机) / 居家模式(智能随机) |
| V1.0 | 📷 拍照识别 | 热量记录 | 拍照识别菜品热量、饮食记录、每日热量汇总 |
| V1.0 | ⏱ 轻断食 | 断食 | 16+8 计时与打卡 |
| V1.0 | 👤 我的 | 个人中心 | 个人信息、目标与设置 |
| V1.5 | 👥 好友 | 社交 | 好友互勉、"已成为好友xx天"、催打卡、排行/组队 |

## 3. 详细需求

### 3.1 模块一 · 今天吃什么

进入模块先选模式（顶部大切换）：**🛵 外卖 / 🏠 居家**。设计原则：外卖极简、居家智能。

#### 3.1.1 外卖模式（纯随机）
| 编号 | 需求 |
|---|---|
| M1-1 | 一个大按钮「摇一摇/换一个」，随机弹出一道菜或品类 |
| M1-2 | 结果仅显示菜名、图、口味；不做任何口味/热量筛选 |
| M1-3 | 操作：换一个 / 就吃这个 |
| M1-4 | 「就吃这个」可跳转外卖平台或"附近能点"（后期） |
| M1-5 | 一次会话内"换一个"不立即重复 |

> 目标：3 秒出结果，绝不加复杂度。

#### 3.1.2 居家模式
**A. 智能随机推荐**
| 编号 | 需求 |
|---|---|
| M1-6 | 随机一道"可在家做"的菜 |
| M1-7 | 推荐考虑：用户口味/菜系偏好、当日剩余热量、忌口过敏、近几天不重复 |
| M1-8 | 轻量筛选：耗时、荤素、菜系 |
| M1-9 | 结果卡：菜名、图、热量、耗时、难度、主要食材 |
| M1-10 | 操作：换一个 / 就吃这个(进详情) / 加入想吃 |

**B. 食材选菜（我有什么→能做什么）**
| 编号 | 需求 |
|---|---|
| M1-11 | 勾选/搜索现有食材，常用置顶 |
| M1-12 | 结果分档：食材齐可直接做 / 缺1–2样(标出缺料) |
| M1-13 | 排序：匹配度 / 热量 / 耗时 |
| M1-14 | 匹配度=已有必需食材/全部必需食材；主料缺失惩罚>调料(调料默认常备) |
| M1-15 | 缺料一键加入购物清单 |

**C. 辅助**
- 菜谱详情页：热量、营养、食材(标缺料)、分步做法、收藏/加想吃/记为某餐/分享
- 想吃清单：随机"想吃没吃"沉淀，下次可"优先从想吃里抽"
- 购物清单：缺料自动汇总，勾选采买、可分享

---

### 3.2 模块二 · 拍照识别热量
| 编号 | 需求 |
|---|---|
| M2-1 | 拍照/相册上传，识别菜品名称+估算热量 |
| M2-2 | 返回：菜名、热量(kcal)、碳水/蛋白质/脂肪 |
| M2-3 | 识别不准可手动修正/搜索；支持一次多菜识别 |
| M2-4 | 结果按 早/午/晚/加餐 记入饮食记录 |
| M2-5 | 当日热量汇总 vs 目标热量(进度条) |
| M2-6 | 历史记录：日/周/月热量曲线 |
| M2-7 | 每日热量目标由个人信息(身高体重目标)自动计算(BMR/TDEE) |

---

### 3.3 模块三 · 16+8 轻断食
| 编号 | 需求 |
|---|---|
| M3-1 | 设定进食窗口(如12:00–20:00)，其余为断食 |
| M3-2 | 大圆环倒计时，显示"断食中/进食中"+剩余时间 |
| M3-3 | 一键"开始进食/结束进食"，自动记录时长 |
| M3-4 | 多模式：16+8 / 14+10 / 18+6 / 自定义 |
| M3-5 | 每日完成自动打卡，累计连续天数(streak) |
| M3-6 | 断食日历(完成/中断可视) |
| M3-7 | 阶段成就徽章(连续7/30/100天) |
| M3-8 | 与饮食记录联动：进食窗口外记录时提醒 |

---

### 3.4 模块四 · 好友互勉（V1.5，非 V1.0 首发）
| 编号 | 需求 |
|---|---|
| M4-1 | 添加好友(手机号/二维码/邀请码) |
| M4-2 | 好友卡片显示 **"已成为好友 xx 天"**(成为好友当天累计) |
| M4-3 | 好友主页：断食连续天数、今日是否打卡、近期动态 |
| M4-4 | 一键"催打卡/点赞/加油" |
| M4-5 | 组队/监督组：一起坚持N天，有人中断全组可见 |
| M4-6 | 动态流：好友完成断食、达成成就时提醒 |
| M4-7 | 排行榜：连续天数 / 本周达标率 |

---

### 3.5 模块五 · 我的（个人信息）
| 编号 | 需求 |
|---|---|
| M5-1 | 资料：头像、昵称、性别、生日、身高、体重、目标体重 |
| M5-2 | 目标类型：减脂/增肌/保持/健康饮食 |
| M5-3 | 每日热量目标、断食模式偏好、忌口与过敏 |
| M5-4 | 数据统计：体重曲线、热量趋势、断食完成率、连续打卡、成就墙 |
| M5-5 | 设置：提醒(进食窗口/记录)、隐私(对好友可见范围)、账号、通知 |

## 4. 信息架构

```
V1.0 底部导航（4 个一级入口）
├─ 今天吃什么
│   ├─ 外卖模式(纯随机)
│   └─ 居家模式：随机推荐 → 菜谱详情
├─ 拍照识别 → 识别结果 → 饮食记录
├─ 轻断食 → 计时主页 / 打卡
└─ 我的 → 资料编辑 / 目标 / 设置

V1.5 增加
├─ 好友 → 好友列表(含xx天) → 好友主页 / 排行 / 组队
├─ 食材选菜 → 缺料 → 购物清单
└─ 成就 / 提醒系统 / 数据周报
```

---

## 5. 数据模型（关键字段）

- **用户 User**：id、昵称、性别、生日、身高、体重、目标体重、目标类型、每日热量目标、断食模式、偏好菜系[]、忌口[]、过敏[]、常备食材[]
- **菜谱 Dish**：id、菜名、图、模式标签(外卖可点/居家可做/皆可)、菜系、口味标签、热量、碳水/蛋白/脂肪、耗时、难度、份量、步骤、必需食材[]、可选食材[]
- **食材 Ingredient**：id、名称、分类、是否调料常备
- **饮食记录 MealLog**：id、用户id、菜名、热量、营养、餐次、时间、图片、是否窗口内
- **断食记录 FastLog**：id、用户id、日期、模式、开始/结束时间、时长、是否完成
- **好友关系 Friend**：id、用户A、用户B、成为好友日期(算天数)、状态
- **互勉记录 Cheer**：id、发起人、接收人、类型(加油/催打卡)、时间
- **想吃/购物清单**：用户id、dishId/食材、类型、数量、来源、是否已买

---

## 6. 模块联动
- 拍照识别 → 饮食记录 + 当日热量
- 饮食记录 → 判断是否在断食进食窗口内
- 居家"就吃这个/记为某餐" → 饮食记录；缺料 → 购物清单
- 个人信息(身高体重目标) → 每日热量目标 → 影响居家推荐与热量提示
- 断食打卡/达标 → 好友动态 + 成就徽章
- 外卖模式保持独立，不联动热量

---

## 7. 变现设计
1. **会员订阅**：去广告、高级菜谱、更准识别、更多断食模式
2. **电商导流**：购物清单跳转生鲜、外卖"附近能点"分成
3. **品牌合作**：菜谱/食材软性植入(标注广告，克制使用)

---

## 8. 版本规划

**V1.0 MVP（跑通核心闭环）**
- 一级入口固定为今天吃什么、拍照识别、轻断食、我的，共 4 个；不放置好友占位入口
- 今天吃什么：模式切换 + 外卖纯随机 + 居家随机推荐 + 菜谱详情
- 拍照识别 + 饮食记录 + 每日热量目标
- 16+8 断食计时与打卡
- 个人信息

**V1.5（加留存）**
- 食材选菜 + 缺料 + 购物清单
- 好友互勉、"xx天"、催打卡、排行/组队
- 成就徽章、提醒系统、数据周报

**V2.0（拓展变现）**
- 会员体系、外卖/生鲜导流、居家智能权重(口味/热量/不重复)、想吃清单优先抽

---

## 9. 非功能性需求（补充）
- 性能：外卖随机结果 ≤1 秒返回；拍照识别 ≤3 秒
- 隐私：健康数据本地优先，好友可见范围用户可控
- 兼容：主流 iOS / Android 机型
- 离线：随机推荐、断食计时支持无网使用
